📖 Theory: Reverse Strategy

Чтобы развернуть список, мы не переставляем узлы в памяти. Мы меняем направление связей.

  • prev: Хранит уже развернутую часть списка (изначально NULL).
  • curr: Текущий узел, связь которого мы "отрываем" от будущего и цепляем к прошлому.
  • next: Временный "сейф", чтобы не потерять ссылку на остаток списка.

Complexity:
Time: O(n) — один проход по списку.
Space: O(1) — используем только 3 указателя.

ListNode* reverseList(ListNode* head) {
    ListNode* prev = nullptr;
    ListNode* curr = head;

    while (curr != nullptr) {
        // 1. Сохраняем следующий узел
        ListNode* nextTemp = curr->next;
        
        // 2. Разворачиваем указатель
        curr->next = prev;
        
        // 3. Сдвигаем prev и curr вперед
        prev = curr;
        curr = nextTemp;
    }
    return prev;
}
